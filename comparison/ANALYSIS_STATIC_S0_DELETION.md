## 一、static_s0 也有窗口推移

### 1.1 static_s0 的实际行为

让我们看看 static_s0 在窗口链中的实际执行：

```
Step 0: 用 S0 + 历史W0 → 预测 W1
Step 1: 用 S0 + 历史W1 → 预测 W2  ← 历史窗口变了！
Step 2: 用 S0 + 历史W2 → 预测 W3  ← 历史窗口又变了！
Step 3: 用 S0 + 历史W3 → 预测 W4
Step 4: 用 S0 + 历史W4 → 预测 W5
```

**关键点**：
- ✅ 画像 S0 是静态的（不变）
- ✅ 但**历史窗口是动态的**（W0 → W1 → W2 → W3 → W4）
- ✅ 所以模型输入**也在变化**

### 1.2 三窗口统计对 static_s0 同样适用

对于 static_s0 的 Step 1：

```python
# 过去窗口：用 S0 + 历史W0 → 预测 W1
past_window_eval = evaluate(S0, history=W0, target=W1)

# 当前窗口：用 S0 + 历史W1 → 预测 W2
current_window_eval = evaluate(S0, history=W1, target=W2)

# 未来窗口：用 S0 + 历史W2 → 预测 W3
future_window_eval = evaluate(S0, history=W2, target=W3)
```

**结论**：static_s0 的三个窗口评估**完全有意义**，因为历史输入在变化。

## 二、static_s0 是最重要的基线

### 2.1 为什么需要 static_s0？

static_s0 是**最基础的基线**，用于回答：

**核心问题**：画像更新是否真的有用？

- 如果 clasp_online ≈ static_s0：画像更新**无效**
- 如果 clasp_online > static_s0：画像更新**有效**

### 2.2 对比价值

| 对比 | 回答的问题 |

**static_s0 是唯一的"零更新"基线**，其他方法都有某种形式的更新或增强。

## 三、你可能混淆的概念

### 3.1 静态画像 ≠ 静态输入

- **静态画像**（static profile）：画像 S0 不变
- **静态输入**（static input）：整个模型输入不变

static_s0 只保证**画像静态**，但**历史窗口是动态的**。

### 3.2 真正的"静态输入"方法

如果你想要一个**完全静态输入**的基线，那应该是：

```python
# 假设的 "truly_static" 方法
Step 0: 用 S0 + 历史W0 → 预测 W1
Step 1: 用 S0 + 历史W0 → 预测 W2  ← 历史也不变
Step 2: 用 S0 + 历史W0 → 预测 W3  ← 历史也不变
...
```

但这个方法**没有意义**，因为：
- 预测 W2 时不知道 W1 发生了什么
- 违反了时间序列预测的基本假设

---

## 四、三窗口统计的真正含义

### 4.1 三窗口统计的目的

三窗口统计**不是为了保持输入一致**，而是为了：

1. **评估泛化能力**：画像在不同时间点上的表现
2. **检测过拟合**：是否只在当前窗口上表现好
3. **检测遗忘**：是否丢失了历史信息

### 4.2 所有方法都适用

三窗口统计对**所有方法**都适用，包括：

- **static_s0**：评估静态画像在不同时间点的表现
- **clasp_online**：评估更新后画像的泛化能力
- **prefix_refresh**：评估重算画像的时间一致性

### 1.2 当前记录的指标

**用户级别**:
- `user_id`: 用户 ID
- `community_id`: 社区 ID
- `method`: 评估方法名称
- `mean_Q`: 所有步骤的平均 Q 值
- `mean_F`: 所有步骤的平均 F 值

**步骤级别** (每个 step):
- `step_index`: 步骤索引（0-4）
- `history_window`: 历史窗口（W0-W4）
- `target_window`: 目标窗口（W1-W5）
- `F`: 交互决策 F1 分数
- `L`: 内容语义相似度
- `Q`: 综合得分

### 1.3 缺失的关键信息

**问题**: 当前代码**没有记录画像更新过程**，无法追踪：
- 每步使用的画像内容
- 候选画像的评分
- 画像是否被更新
- 更新前后的性能对比

## 二、证明方法有效性的统计需求

### 2.1 核心问题

要证明 **clasp_online 的画像精炼机制有效**，需要回答：

1. **画像是否在改进？** 
   - 更新后的画像是否比更新前表现更好？

2. **改进是否持续？**
   - 随着时间推移，画像质量是否持续提升？

3. **改进体现在哪里？**
   - F（决策准确性）提升？
   - L（内容相似度）提升？
   - 还是两者都提升？

### 2.2 时间轴分析维度

#### 维度 1: 步骤维度（Step-wise）
- **横轴**: step_index (0, 1, 2, 3, 4)
- **纵轴**: F / L / Q 指标
- **目的**: 观察随时间推移，性能是否提升

#### 维度 2: 窗口维度（Window-wise）
- **过去窗口**: 用于生成初始画像（W0）
- **当前窗口**: 用于预测和精炼（W1-W4）
- **未来窗口**: 用于验证（W2-W5）

#### 维度 3: 画像更新维度（Profile Update）
- **更新前**: 使用旧画像的性能
- **更新后**: 使用新画像的性能
- **增益**: Δ = 更新后 - 更新前

## 三、统计方案设计

### 3.1 增强数据记录（修改代码）

需要在 `window_chain_eval.py` 中记录以下额外信息：

```python
# 每个 step 增加字段
{
  "step_index": 0,
  "history_window": "W0",
  "target_window": "W1",
  "F": 0.6234,
  "L": 0.4521,
  "Q": 0.5547,
  
  # === 新增字段 ===
  "profile_updated": true,              # 画像是否被更新
  "profile_length": 1234,               # 当前画像长度（字符数）
  "num_candidates": 10,                 # 生成的候选画像数
  "best_candidate_index": 3,            # 选中的候选索引
  
  # 候选画像评估（仅 clasp_online）
  "candidate_scores": [
    {"index": 0, "Q": 0.5234, "F": 0.6012, "L": 0.4123},
    {"index": 1, "Q": 0.5456, "F": 0.6234, "L": 0.4321},
    ...
  ],
  
  # 画像更新前后对比（在下一步验证）
  "next_step_with_old_profile": {
    "F": 0.6100,  # 如果继续用旧画像的预测结果
    "L": 0.4400,
    "Q": 0.5420
  },
  "next_step_with_new_profile": {
    "F": 0.6234,  # 实际用新画像的预测结果
    "L": 0.4521,
    "Q": 0.5547
  },
  "profile_update_gain": {
    "ΔF": 0.0134,
    "ΔL": 0.0121,
    "ΔQ": 0.0127
  }
}
```

### 3.2 统计指标设计

#### 指标 1: 画像更新率（Profile Update Rate）
```
更新率 = 实际更新次数 / 总步骤数
```
- 衡量画像精炼的活跃程度
- 如果更新率很低，说明画像很快就稳定了

#### 指标 2: 平均更新增益（Average Update Gain）
```
平均增益 = Σ(Q_new - Q_old) / 更新次数
```
- 衡量每次更新带来的平均性能提升
- 正值表示更新有效，负值表示更新有害

#### 指标 3: 累积增益（Cumulative Gain）
```
累积增益 = Q_final - Q_initial
```
- 衡量从初始画像到最终画像的总体提升
- 对比 static_s0（不更新）的基线

#### 指标 4: 步骤间改进率（Step-wise Improvement Rate）
```
改进率(step_i) = (Q_step_i - Q_step_0) / Q_step_0
```
- 衡量相对于初始步骤的相对改进
- 观察改进是否随时间递增

#### 指标 5: 候选画像质量分布（Candidate Quality Distribution）
```
- 最佳候选 Q 值
- 最差候选 Q 值
- 候选 Q 值标准差
```
- 衡量候选画像的多样性和质量范围
- 高标准差表示候选差异大

## 五、统计分析方法

### 5.1 描述性统计

**全局统计**:
```python
{
  "total_users": 1801,
  "total_steps": 9005,  # 1801 × 5
  "total_updates": 7234,
  "update_rate": 0.803,
  "mean_update_gain": 0.0234,
  "std_update_gain": 0.0156
}
```

**分社区统计**:
```python
{
  "community_0": {
    "users": 500,
    "mean_Q_step_0": 0.5234,
    "mean_Q_step_4": 0.5678,
    "cumulative_gain": 0.0444,
    "update_rate": 0.812
  },
  ...
}
```

**分步骤统计**:
```python
{
  "step_0": {
    "mean_Q": 0.5234,
    "std_Q": 0.0876,
    "mean_F": 0.6123,
    "mean_L": 0.4012
  },
  ...
}
```

### 5.2 假设检验

**检验 1: 画像更新是否显著提升性能？**
```
H0: μ(Q_new) = μ(Q_old)
H1: μ(Q_new) > μ(Q_old)
方法: 配对 t 检验
```

**检验 2: clasp_online 是否优于 static_s0？**
```
H0: μ(Q_clasp) = μ(Q_static)
H1: μ(Q_clasp) > μ(Q_static)
方法: 独立样本 t 检验
```

**检验 3: 性能是否随步骤提升？**
```
H0: 各步骤 Q 值无差异
H1: 至少一个步骤显著不同
方法: 重复测量 ANOVA
```

### 5.3 相关性分析

**分析 1: 初始性能 vs 改进幅度**
```
问题: 初始表现差的用户是否改进更多？
方法: Pearson 相关系数
```

**分析 2: 社区特征 vs 更新效果**
```
问题: 不同社区的更新效果是否不同？
方法: 方差分析（ANOVA）
```
